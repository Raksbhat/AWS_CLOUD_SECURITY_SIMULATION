Detection Engine — EventBridge

Amazon EventBridge listens to CloudTrail events in real time. When a risky API call occurs, EventBridge rules filter and route it to the correct Lambda remediation function — automatically, within seconds.

🪣 S3 EXPOSURE EVENTS
  • PutBucketPolicy
  • PutBucketAcl
  • DeletePublicAccessBlock
🔑 IAM CHANGE EVENTS
  • AttachRolePolicy
  • PutRolePolicy
  • CreateAccessKey
🛡️ SECURITY GROUP EVENTS
  • AuthorizeSecurityGroupIngress
  • 0.0.0.0/0 on port 22 (SSH)
  • 0.0.0.0/0 on port 3306 (MySQL)
// 03
Automated Remediation — Lambda

Each Lambda function (Python) handles one specific remediation action. Tag-based governance prevents automation from accidentally breaking legitimate configurations.

🪣
S3 PUBLIC BUCKET REMEDIATION

If a bucket becomes public, Lambda re-enables Block Public Access — unless tagged intended-public=true.

  • 1Extract bucket name from CloudTrail event
  • 2Check bucket tags for intended-public=true
  • 3Enable BlockPublicAcls + IgnorePublicAcls
  • 4Enable BlockPublicPolicy + RestrictPublicBuckets
🛡️
SECURITY GROUP REMEDIATION

If a security group allows 0.0.0.0/0 on port 22 (SSH) or 3306 (MySQL), Lambda revokes only the unsafe rule.

  • 1Identify the security group ID from event
  • 2Extract the risky ingress rule details
  • 3Call ec2:RevokeSecurityGroupIngress
  • 4Log the action to centralized S3 bucket
🔑
IAM PRIVILEGE MONITORING

Detects unexpected high-privilege IAM policy attachments. Logs the event, sends an SNS alert, and enables investigation.

  • 1Detect AttachRolePolicy / PutRolePolicy
  • 2Log event to immutable audit S3 bucket
  • 3Publish SNS alert to security team
  • 4Flag for incident investigation
🏷️
TAG-BASED SECURITY GOVERNANCE
Tag intended-public=true on any resource to exempt it from auto-remediation. This prevents breaking legitimate public configs (e.g. public websites) while still protecting unintended exposures. This pattern is widely used in enterprise production AWS environments.

// 05
Security Principles
🔒
LEAST PRIVILEGE
Each Lambda gets only the exact permissions needed — s3:PutBucketPublicAccessBlock, ec2:RevokeSecurityGroupIngress. Nothing more.
📜
IMMUTABLE LOGGING
S3 Object Lock + Versioning + KMS Encryption. Attackers cannot delete or modify audit trails even if production is compromised.
⚡
EVENT-DRIVEN
Responses triggered by real infrastructure events, not scheduled scans. Real-time detection with sub-5-second remediation.
🏗️
MULTI-ACCOUNT ISOLATION
Production, Logging, and Deployment are separate AWS accounts. A compromise in one cannot easily spread to others — limited blast radius.
🏷️
TAG GOVERNANCE
Tag-based exemptions prevent false positives. intended-public=true skips remediation for approved resources.
🔄
INFRASTRUCTURE AS CODE
All resources deployed via Terraform — reproducible, auditable, version-controlled. State locked with DynamoDB, encrypted with KMS.
