// 01 Multi-Account Architecture
🏭
Production Account
PROD

Hosts all live infrastructure. AWS CloudTrail records every API call — IAM changes, S3 policy updates, VPC security group modifications. EventBridge routes risky events to Lambda for instant remediation.

AWS IAM S3 Buckets VPC Security Groups CloudTrail Amazon EventBridge Lambda
📦
Logging Account
LOG

Centralized, tamper-proof log storage. Receives org-wide CloudTrail logs. S3 Object Lock + Versioning ensures logs are immutable. Even if production is compromised, audit logs remain safe.

Org CloudTrail Logs S3 Versioning S3 Object Lock AWS KMS Encryption Centralized S3 Bucket Lambda (Remediation)
🚀
Deployment Account
DEPLOY

Manages all Infrastructure as Code deployments via Terraform. Uses STS AssumeRole for cross-account access. CI/CD pipeline deploys securely without direct production access.

Terraform Terraform State S3 DynamoDB State Lock AWS KMS STS AssumeRole CI/CD Pipeline

// 08 Security Principles Implemented
🔒
Least Privilege Access
Each Lambda function only has the exact permissions needed for its remediation action — nothing more.
🗂️
Immutable Logging
S3 Object Lock + Versioning ensures audit logs can never be deleted or modified — even by attackers.
⚡
Event-Driven Security
Security responses triggered by infrastructure events in real-time — no manual monitoring required.
🏢
Multi-Account Isolation
Production, Logging, and Deployment are separated accounts — limiting the blast radius of any compromise.
🏷️
Tag-Based Governance
Approved resources use tags to opt-out of automation — preventing false positives on legitimate configs.
🏗️
Infrastructure as Code
All resources deployed via Terraform — reproducible, auditable, version-controlled infrastructure.
